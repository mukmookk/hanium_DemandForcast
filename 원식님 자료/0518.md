## Goal
----
만약에 한 달 단위로 한다면 `3월`,`4월`,`5월`,`6월`의 데이터가 있고, `6월` 이후 `나머지 月의 수요를 예측`하는 것을 잠정적인 목표로 설정

# Problem & Idea
----

지금 컬럼으로는 뭔가 지표같은게 부족

-> 파생변수를 생성해보자

### 파생 변수의 예시
```
3월의 score는 = (3月 품목금액 % 3月 품목수량) % 日에 루트 (여기서 품목금액이 품목수량보다 적을 수 없음)

6월 = (49612988285 % 1372236) % 30 에 루트 = 34.71543 (6_SCORE)
```

`해당 수치가 감소한다 = 품목수량이 상승하는 月`이라 생각할 수 있다.

따라서 그게 감소하는 `월(y)`을 예측함.

`score`를 독립변수로 쓸지,  `y`로 쓸지는 별도의 회의가 필요하다.

제시한 파생 변수를 이렇게 해석할 수 있을 거 같다.
```
품목금액이 큰데 수량이 작다 -> 비싼게 팔린거 (판매수량이 적은거라고 생각가능)

하지만 금액은 작은데 수량이 많다 -> 판매수량이 많은 것.

결국 금액대비 수량이 높다는건 주문량이 많다는 걸 의미한다.

동일한 20만원수익에 % 10개판매 = 20,000
동일한 20만원수익에 % 100개판매 = 2000
```

따라서 `금액/주문량 했을때 숫자가 낮을수록 많이 팔린 것 = (주문량이 많은거)`

숫자가 크기에 `月`의 일수로 나눈 후 루트를 씌우면 값이 지표로 쓰기에 값이 적당하다고 생각.

```
ex) 5월 한달 품목금액이 3천만원 품목수량은 7500개라고 가정. 
(30,000,000 / 7500) = 4000  	
4000 % 30 = 133 
루트133 = 11.54
품목수량이 9000개 팔렸으면 10.54
품목수량이 12000개 팔렸으면 9.12
값이 감소하는 것을 확인.
```

판매량 증가 할수록 파생변수가 감소하는 것 대신에 판매량과 비례로 상승할 수 있도록 하는 파생변수가 있다면 더 좋을 것이라 생각 

--------------------

`월 별`로 위 `스코어 값`을 도출
```
ex)
3월 = 12
4월 = 15
5월 = 17
6월 = 20
7월 = ?
```

### 추가적인 아이디어
----

이때 7월을 예측하면 역순으로 계산해서 주문량을 대충 알 수 있지 않을까?
예시라서 한달 단위이지만 만약 한달을 더 쪼갤 수 있으면 좋음 (한달의 상/하반기) or 주 단위

3,4,5,6월의 독립변수들을 생각해서 회귀를 돌려 모형을 만들면?
다음달의 저 스코어를 예측하는 모델을 만들 수 있음.

문제점은 독립변수로 쓸 컬럼이 마땅히 없고, 이런 점수를 줄 수 있는(가중치도 괜찮음) 파생변수를 많이 생각해내면 좋을 것 같음
